# Product scope

## Working name

JENN Programme Intelligence System

## Core thesis

JENN teaches focused Grade 12 intervention sessions. The system exists to capture the operational evidence and turn it into clear summaries for leadership and department officials.

## Main workflow

1. Create a programme.
2. Register learners into the programme.
3. Create daily sessions by subject, topic, time and venue.
4. Capture attendance for every session.
5. Capture facilitator notes after each session.
6. Generate programme dashboards and department-ready reports.

## Primary users

- JENN admin: creates programmes and monitors everything.
- Facilitator: marks attendance and captures session notes.
- Viewer / department official: views dashboards and reports.

## MVP boundaries

Included now:

- dashboard
- programme control
- learner registration shell
- session schedule
- attendance capture shell
- facilitator note shell
- department report view
- Supabase schema
- demo mode

Not included yet:

- payments
- parent accounts
- AI summaries
- mobile app
- WhatsApp/SMS
- advanced permissions
- final JENN branding
- live exports beyond print/PDF view

## Reporting questions answered

- How many learners registered?
- Which schools are represented?
- How many sessions were scheduled?
- Which subjects and topics were covered?
- What was the attendance rate?
- Which learners/schools need follow-up?
- What did facilitators observe?
- What should be done after the intervention?

## Strategic positioning

This is separate from LearnSphere.

LearnSphere is the learner/parent academic accountability product.

This system is an institutional programme-monitoring product.

They may connect later, but they should not share code or database logic at this stage.
